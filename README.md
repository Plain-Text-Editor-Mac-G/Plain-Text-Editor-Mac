## 📖 Overview

<a href="#plainTextEditor">Plain Text Editor</a> serves writers who want a clean writing environment without rich text distractions, developers who need a quick plain text scratch pad, anyone who regularly pastes formatted text and needs it stripped, and Mac users who want a reliable plain text file editor without TextEdit's rich text complexity.

The <a href="#plainTextEditor">formatting removal use case</a> is the most immediate practical value — copying text from a web page, email, or word processor document and pasting it into Plain Text Editor instantly strips all formatting, producing clean plain text that can be copied again for use in contexts that require it. This use case alone — paste formatted, copy plain — makes Plain Text Editor a daily utility for knowledge workers who work across multiple content sources. The <a href="#plainTextEditor">developer scratch pad use case</a> uses Plain Text Editor as a quick note-taking and temporary text editing space — pasting API responses, configuration snippets, log output, and other plain text content for quick review and editing without opening a heavy code editor or a formatted document.

The <a href="#plainTextEditor">focused writing environment</a> serves writers who specifically want to write without the formatting affordances that word processors and rich text editors provide — no bold, no headers, no bullet points that might distract from pure prose writing. Some writers find that the availability of formatting in their editor creates a temptation to format rather than write, and Plain Text Editor removes that entirely. <a href="#plainTextEditor">Free availability from Mac App Store</a> — Plain Text Editor is free to download, consistent with Sindre Sorhus's approach to making high-quality small utilities freely available.

<a href="#plainTextEditor">Automatic plain text file handling</a> opens and saves .txt files correctly — double-clicking a .txt file in Finder opens it in Plain Text Editor, and saving produces a plain .txt file without hidden formatting metadata. <a href="#plainTextEditor">System Services integration</a> enables right-click conversion of selected text in other applications through macOS Services — converting rich text to plain text from any application without opening Plain Text Editor directly.

<a href="#plainTextEditor">iCloud document sync</a> stores documents in iCloud for access across Mac devices — plain text files written on one Mac available on another through the standard iCloud Documents integration. <a href="#plainTextEditor">Regular macOS compatibility updates</a> from Sindre Sorhus ensure Plain Text Editor continues working correctly through macOS updates.

## 🤔 FAQ

<details>
<summary>Can I use Plain Text Editor to convert rich text to plain text in other apps?</summary>

Yes. System Services integration enables right-click conversion of selected text in other applications through macOS Services, without opening Plain Text Editor directly.

</details>

<details>
<summary>Can I change the font size in Plain Text Editor?</summary>

Yes. Font size adjusts to match reading and writing preferences without affecting the saved file content.

</details>

<details>
<summary>Does Plain Text Editor use a monospaced font?</summary>

Yes. Monospaced font is the default display typeface for Plain Text Editor.

</details>

<details>
<summary>Does Plain Text Editor run on Apple Silicon?</summary>

Yes. Current versions support Apple Silicon Macs.

</details>

<details>
<summary>Can I use Plain Text Editor for code editing?</summary>

Plain Text Editor is suitable for viewing and editing code snippets in plain text. For full-featured code editing with syntax highlighting and language tools, a dedicated code editor is more appropriate.

</details>

<details>
<summary>What is the difference between Plain Text Editor and TextEdit in plain text mode?</summary>

Both edit plain text files. Plain Text Editor is entirely plain text with no rich text mode, providing a simpler and more focused experience than TextEdit which includes rich text mode and more interface complexity.

</details>
